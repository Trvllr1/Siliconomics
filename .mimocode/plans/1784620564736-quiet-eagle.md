# Siliconomics Website Redesign — Implementation Plan

**Goal:** Transform the existing marketing website from a functional skeleton into a world-class, Fortune-100-boardroom-credible presentation that passes The Manhattan Test on every page.

**Current state:** All routes, pages, components, and content modules exist. The architecture (router, engine access, content system) is solid. The gaps are in visual fidelity, interactive polish, accessibility, and SEO.

**Constraints (inviolable):**
- `/app` subtree unchanged. All 93 existing Vitest tests pass untouched.
- All numbers trace to engine output or governing documents.
- No fabricated logos, testimonials, or usage statistics.
- Demo mode stays ungated forever.

**Approved new dependency:** `katex` — for mathematical typesetting on the Methodology page.

**Design decisions (confirmed with owner):**
- Product visuals: CSS/SVG wireframe representations (self-contained, accurate, dark-terminal aesthetic)
- Equations: KaTeX for proper mathematical typesetting with version tags
- Build scripts: Extend existing `generateOgImages.mjs` and `generateSitemap.mjs`

## Phase 0 — Visual Identity & Global Foundation

**Goal:** Establish the atmospheric "Dark Terminal" treatment that makes every page feel like Bloomberg Terminal meets executive briefing center.

### 0.1 Blueprint Grid Texture & Radial Vignette

**`src/index.css`** — Add two CSS utility classes:
- `.blueprint-grid`: `repeating-linear-gradient` with `ink @ 3%`, 64px pitch — the architectural texture

### 0.2 Navigation Overhaul

**`src/site/components/SiteNav.tsx`** — Major rewrite:
- Add `⌘K` search hint chip (styled `<kbd>` element) in the center nav area, wired to open CommandPalette
- Add CommandPalette state management (`open` state, keyboard listener for `⌘K`/`Ctrl+K`)
- Add **scroll-linked progress hairline** — thin `art-rust` line at top of nav that fills based on scroll position (move from Methodology page to nav for consistency across all long-form pages)
- Mobile drawer: replace raw div toggle with animated slide-over, add `aria-modal="true"`, focus trap, `Escape` to close
- Add visible focus rings on all interactive elements: `focus-visible:ring-2 focus-visible:ring-art-rust focus-visible:ring-offset-2`
- Mobile: ensure Launch Demo button is always visible

### 0.3 SEO Component

**`src/site/components/SEO.tsx`** — New file:
- Reusable component using `document.title` and `<meta>` tag manipulation (no new dependency — use `useEffect` to set head elements)

### 0.4 Skip Link & Accessibility Baseline

**`src/site/pages/Home.tsx`** — Add skip link (`<a href="#main-content" className="skip-link">Skip to main content</a>`) since Home doesn't use SiteLayout

**All components** — Add ARIA attributes:

## Phase 5 — SEO, Performance & Final Audit

**Goal:** Lighthouse ≥ 95 all categories. Zero axe violations. OG previews work.

### 5.1 SEO Implementation

**Every page file** — Add `<SEO>` component with page-specific metadata:
- `<title>` and `<meta name="description">` per page
- Open Graph tags: `og:title`, `og:description`, `og:image`, `og:type`
- Canonical URLs
- `robots` meta for 404 (noindex)

**`index.html`** — Head overhaul:
- `<meta name="theme-color" content="#0D1117">`
- Font preloads with `display=swap`
- Preconnects for Google Fonts
- Structured data: `Organization` + `SoftwareApplication` JSON-LD on Home

### 5.2 OG Image Generation

**`scripts/generate-og-images.mjs`** — Build-time script:
- Renders per-page OG image template: Fraunces headline + JetBrains Mono metric + provenance stamp on `#0D1117`
- Uses `node-canvas` or `sharp` (existing script at `scripts/generateOgImages.mjs` — verify and extend)
- Output: `public/og/*.png` — one per top-level route

### 5.3 Sitemap & Robots

**`scripts/generate-sitemap.mjs`** — Build-time script:
- Generates `public/sitemap.xml` from route definitions
- Generates `public/robots.txt` referencing sitemap
- Existing script at `scripts/generateSitemap.mjs` — verify and extend

### 5.4 Performance Verification

**`vite.config.ts`** — Verify chunk strategy:
- Marketing routes must NOT load `vendor-clerk`, `vendor-query`, `vendor-charts`
- First-party JS on marketing routes < 150KB gzipped
- Verify lazy loading works for all marketing pages

### 5.5 Accessibility Audit

**axe-core integration:**
- Run `axe-core` against every page component
- Zero violations required
- Full keyboard walkthrough: tab through every page, verify focus visibility, logical order, no traps

### 5.6 Reduced Motion Verification

**All animated components:**
- Verify `prefers-reduced-motion: reduce` produces fully legible, non-animated pages
- LiveTerminal: shows static output without typing animation
- ConsequenceExplorer: shows default values without tick-over
- ProgramChart: renders without entrance animation
- All scroll reveals: content visible without fade-in

### 5.7 No-JS Fallbacks

**Critical components:**
- LiveTerminal: `<noscript>` block with complete terminal output
- ProgramChart: hidden `<table>` with chart data
- ConsequenceExplorer: static snapshot of default values

### 5.8 Final Verification

**Exit checks:**
1. All 93 existing Vitest tests pass
2. Application at `/app` is functionally identical
3. Lighthouse ≥ 95 on all marketing routes
4. axe-core: zero violations
5. Every displayed number traces to engine output
6. OG/Twitter previews render on all top-level routes
7. Responsive 360px → 2560px without layout debt
8. `prefers-reduced-motion` verified on all pages
9. No fabricated logos, testimonials, or metrics
10. Demo mode stays ungated
